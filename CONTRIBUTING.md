# 🤝 Contributing Guidelines

- Follow clean Apex and Flow best practices
- Maintain proper naming conventions
- Include test classes for Apex contributions
- Add documentation for new modules
- Avoid posting direct Trailhead answers - focus on learning explanations
- Keep solutions generic and reusable

## 🤝 How to Contribute

1. Fork the repository
2. Create a new branch
  ```bash
  git checkout -b feature/<feature-name>
  ```
3. Make your changes
  - Commit your changes with clear messages
    ```bash
    git commit -m "<commit message>"
    ```
4. Push to your fork
  ```bash
  git push origin feature/<feature-name>
  ```
5. Submit a Pull Request

---

## 🧠 Code Standards

- Bulkified Apex only
- Minimum 75% test coverage
- Avoid hardcoding IDs
- Follow Salesforce naming conventions
- Document assumptions clearly

---

## 🛠️ Issue Reporting

If you find an issue:

1. Open a GitHub Issue
2. Provide:
  - Module name
  - Description of problem
  - Expected behavior
  - Screenshots (if applicable)
